# Time Cap — 開発者向けアーキテクチャメモ

## この版の肝（必読）

**音楽ランキングは「最初のジャンル」にすぎない。**

振り返り（受動）の中核は、**期間まとめ＋シェアカード（Wrapped 方式）の汎用テンプレ**である。  
現状の `musicMonths` / `openWrappedCard()` / 振り返りタブのランキング UI は、そのテンプレの **genre: `music`** 実装として扱う。

### やること

- まとめの仕組みは **1つ** に保つ（期間キー・ランキング・1位ハイライト・縦型シェアカード）
- 新しい痕跡は **ジャンルを足す** だけ（別画面・別集計パイプラインを増やさない）

### やらないこと

- 人・声・ルート・推しを **それぞれ独立実装** しない（工数が膨らみ、プロダクトの焦点がボヤける）
- ジャンルを一度に全部出す（核が 5 秒で伝わらなくなる）

### ジャンル追加の優先順位（P1 以降）

1. `music` — 実装済み（テンプレの参照実装）· マップ画面の「音楽」サブタブ
2. `route` — **思い出マップ**（軌跡の線＋点 · Exif のみ · シェアカード）— 実装中
3. `people` — よく一緒にいた人ランキング
4. `voice` — 30 秒の声（封じる側の能動とも接続可）
5. `fandom` — 推し・ハマりの変遷

各ジャンルは同じデータ形・同じ UI フローに乗せる。表示ラベルと集計ソースだけ差し替える。

---

## データ形（目標）

```js
// 現状: musicMonths[] が先駆。将来は periodSummaries に統合してよい。
{
  key: '2026-06',           // 期間 ID（月 / 週 / 季節）
  label: '2026年6月',
  genre: 'music',           // music | people | voice | route | fandom
  ranking: [{ rank, title, subtitle?, plays?, meta? }],
  highlight: { ... },       // 1位 = テーマソング / その期間の象徴
}
```

`openWrappedCard()` は genre に依存しない **1つのシェアカード生成器** に育てる。  
音楽専用の分岐を増やすのではなく、`highlight` / `ranking` の表示フィールドを汎用化する。

---

## 2 本柱（プロダクト）

| 柱 | モード | 主な UI |
|----|--------|---------|
| A | 封じる（能動） | 待合室・今日の問い・作成シート |
| B | 振り返り（受動） | 期間まとめテンプレ（ジャンル追加） |

機能が増えても、この 2 本柱と **1 テンプレ** を崩さない。

---

## 実装の入口（現コード）

| 役割 | ファイル内の目安 |
|------|------------------|
| 期間まとめ（music） | `defaultMusicMonths()`, `S.musicMonths` |
| 思い出マップ（route） | `initMemoryMap()`, `renderMemoryMap()`, `mapPointsFromOpened()`, `openMapShareCard()` |
| Exif 位置 | `extractGeoFromFile()` → 封じる時 `loc` → 開封後 `opened[].loc` |
| シェアカード | `openWrappedCard()`, `openMapShareCard()`, `#wrappedCard` |
| 日次・週次（補助） | `S.musicToday`, `S.musicWeek` — まとめテンプレとは別レイヤー |
| 下タブ（4 固定） | ホーム · マップ · ＋封じる · 自分 |

新ジャンルを足すときは、上記を **コピペ実装せず** `genre` を増やして同じ render / share パスを通す。
